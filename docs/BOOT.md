# BOOT: Загрузочная последовательность VibeCraft V4.1

## Последовательность загрузки

Перед выполнением любой задачи Исполнитель **обязан** пройти следующие шаги в строгом порядке:

### Шаг 1: Чтение BRIEF
Прочитать `docs/BRIEF.md`. Понять:
- Суть проекта и миссию
- Целевую аудиторию и её боли
- Scope и Out of Scope
- Технологический стек
- Roadmap

### Шаг 2: Чтение RULES
Прочитать `docs/RULES.md`. Запомнить:
- Стек и архитектуру
- Строгие табу (английский текст, `#000`/`#fff`, AI-слоп)
- SEO-правила
- Контекстные ограничения

### Шаг 3: Чтение ENVIRONMENT
Прочитать `docs/ENVIRONMENT.md`. Понять:
- Как запускается проект
- Какие зависимости подключены
- Структуру папок и файлов

### Шаг 4: Чтение LESSONS
Прочитать `docs/LESSONS.md`. Извлечь:
- Системные уроки (LL-PROC-XXX)
- Специфичные уроки проекта (LL-SEO-XXX, LL-UI-XXX)
- Применить релевантные уроки к текущей задаче

### Шаг 5: Взятие задачи
1. Найти файл задачи в `tasks/todo/`
2. Прочитать `<context>`, `<task>`, `<rules>`
3. Проверить зависимости (указанные в ЗАВИСИМОСТИ)
4. Начать выполнение согласно правилам из `RULES.md`
5. По завершении: **обязательно заполнить COMPLETION LOG** в файле задачи
6. Перенести файл задачи из `tasks/todo/` в `tasks/done/`
7. Выполнить `git add -A`, `git commit`, `git push origin main`
