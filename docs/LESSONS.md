# LESSONS: Уроки проекта 66ai.ru

## Системные уроки VibeCraft V4.1

### LL-PROC-001: Атомарность задач
**Категория:** Процесс
**Уровень:** Обязательно

Каждая задача должна затрагивать минимальное количество файлов. Если задача требует изменения более 5 файлов — её нужно разбить на подзадачи. Это обеспечивает чистую историю Git, лёгкий code-review и возможность отката без повреждения всего проекта.

### LL-PROC-002: COMPLETION LOG обязателен
**Категория:** Процесс
**Уровень:** Обязательно

После выполнения задачи **обязательно** заполнить раздел COMPLETION LOG в файле задачи:
- Статус, дату, исполнителя
- Что сделано (списком)
- Изменённые файлы
- Верификация (чеклист)
- Побочные эффекты / риски
- Открытые вопросы

Без заполненного лога задача считается **незавершённой**, даже если код написан.

### LL-PROC-003: Запрет на боевые изменения без задачи
**Категория:** Процесс
**Уровень:** Обязательно

Запрещено изменять HTML, CSS, PHP, API или любые боевые файлы вне контекста задачи из `tasks/todo/`. Каждый коммит должен быть привязан к конкретной таску. Это обеспечивает трассируемость: любой коммит → задача → обоснование.

## Специфичные уроки проекта

### LL-SEO-001: Использование старых доменов в мета-тегах
**Категория:** SEO
**Уровень:** Критично

**Проблема:**
Сайт не индексировался Яндексом, потому что во всех canonical-ссылках и OG-тегах был указан старый домен `ai-fedor.com`. Поисковый робот получал противоречивые сигналы и игнорировал страницы.

**Правило:**
Для данного проекта использовать **ТОЛЬКО** `https://66ai.ru` в следующих местах:
- `<link rel="canonical" href="https://66ai.ru/..." />`
- `<meta property="og:url" content="https://66ai.ru/..." />`
- `<meta property="og:image" content="https://66ai.ru/..." />`
- `<meta property="twitter:image" content="https://66ai.ru/..." />`
- Schema.org JSON-LD поля `url` и `sameAs`
- `sitemap.xml` — все `<loc>` элементы

**Проверка:**
Перед коммитом выполнить: `grep -r "ai-fedor.com" .` — результат должен быть пустым для `.html` и `.xml` файлов.

**Применение:**
Автоматически проверять все новые HTML-файлы и изменения в существующих.
