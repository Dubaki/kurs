# ADR-001: Архитектура системы "Аттестация 66AI"

**Дата:** 2026-04-12
**Статус:** accepted
**Контекст:** Система тестирования корпоративных клиентов через Telegram-бота с визуализацией результатов на сайте 66ai.ru.

---

## Контекст

Компания запускает B2B-услугу «Аттестация 66AI» — многоуровневое тестирование ИИ-компетенций сотрудников. Сотрудники проходят тестирование через Telegram-бота, руководитель получает дашборд с аналитикой, сотрудники — сертификаты и дипломы.

Ключевое требование: вся система должна работать без зависимости от зарубежных сервисов, с учётом РФ-специфики (TenChat, ВКонтакте, hh.ru, Telegram).

---

## Решение (Утверждённые фичи)

### 1. AI Skill Radar
Радарная диаграмма (radar/spider chart) в дашборде руководителя, отображающая компетенции отдела по осям:
- Промпт-инжиниринг
- Этичное использование ИИ
- Автоматизация процессов
- Работа с API/интеграциями
- Генерация кода

**Цель:** Руководитель мгновенно видит сильные и слабые стороны команды.

### 2. Система Верификации
Публичный роут `66ai.ru/verify/{ID}` — страница, куда ведёт QR-код с сертификатов и дипломов.

- `{ID}` — уникальный UUID сертификата
- Страница содержит: ФИО сотрудника, уровень, дату, компетенции (радара)
- Защита от подделки: хеш-подпись в URL (`/verify/{ID}?sig=hash`)

**Цель:** Любой работодатель может проверить подлинность сертификата без регистрации.

### 3. Бенчмаркинг отрасли
Сравнение результатов сотрудника со средними показателями по нише.

- При регистрации указывается отрасль компании (производство, IT, маркетинг и т.д.)
- На дашборде: «Ваш сотрудник на 73% лучше среднего по отрасли»
- Анонимизированные агрегированные данные

**Цель:** Мотивация руководителей через конкурентный контекст.

### 4. Социальный Виджет (РФ)
После прохождения аттестации сотрудник получает виджет «Поделиться достижением» с кнопками:

| Платформа | Механизм |
|---|---|
| **TenChat** | Шеринг поста с изображением сертификата |
| **ВКонтакте** | Share-ссылка через `vk.com/share.php` |
| **Telegram** | Share-кнопка через `t.me/share/url` |
| **HeadHunter (hh.ru)** | Копирование текста для вставки в раздел «Достижения» профиля |

**Цель:** Вирусный рост через социальные сети РФ-сегмента.

## Data Flow (Проект)

```
┌─────────────────────┐
│ 1. Telegram Bot     │  Сотрудник проходит тест
│    (Тест)           │  Бот отправляет вопросы, принимает ответы
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ 2. Бэкенд           │  Сохранение логов, оценок, расчёт баллов
│    (API + DB)       │  Генерация UUID сертификата
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ 3. Генерация PDF    │  Сертификат с QR-кодом (ссылка на /verify/{ID})
│    (Сертификат)     │  Отправка PDF сотруднику через бота
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ 4. Дашборд 66ai.ru  │  API-запрос → визуализация результатов
│    (HR-панель)      │  Radar Chart, бенчмаркинг, аналитика
└─────────┬───────────┘
          │
          ▼
┌─────────────────────┐
│ 5. /verify/{ID}     │  Публичная страница верификации
│    (Страница)       │  Доступна без авторизации, с QR-сканера
└─────────────────────┘
```

## Технический стек (проект)

| Компонент | Технология | Обоснование |
|---|---|---|
| Telegram Bot | Python + aiogram | Зрелая библиотека, async, РФ-сообщество |
| Бэкенд API | PHP (текущий стек) или Python FastAPI | Python предпочтителен для ML/аналитики |
| База данных | PostgreSQL (Supabase) | Бесплатный tier, надёжность, JSON-поля для雷达 |
| PDF-генерация | WeasyPrint / ReportLab | Python, серверная генерация |
| QR-коды | qrcode (Python lib) | Простая интеграция |
| Фронтенд Radar Chart | Chart.js (CDN) | Лёгкая интеграция в текущий HTML-стек |
| Дашборд | Отдельная HTML-страница + API | В стиле текущего сайта |

---

## Риски и митигация

| Риск | Митигация |
|---|---|
| Утечка персональных данных | Хранение только ФИО + email, шифрование, ФЗ-152 |
| Накрутка тестов | Ограничение по времени, рандомизация вопросов, lock-in |
| Зависимость от Telegram | Резервный канал: веб-форма на сайте |
